SPICE 실습 페이지 사용법

학생이 자유롭게 SPICE netlist를 작성·시뮬레이션하는 실습 도구입니다. 시뮬레이션은 브라우저 안의 ngspice WASM으로 실행되며, 서버로 전송되지 않습니다.

1. 기본 사용

  • 왼쪽 editor에 netlist를 작성합니다. 모델은 자동으로 /model.lib에 로드되니 .lib "/model.lib" MOS로 참조합니다.
  • 지원 해석: .tran, .dc, .ac, .noise. 한 netlist에 여러 개 작성 가능.
  • 오른쪽 탭(Tran/DC/AC/Noise)을 선택하고 ▶ Run을 누르면 해당 해석만 ngspice로 실행됩니다.

2. 탭별 독립 실행

각 Run 버튼은 ngspice를 별도 세션으로 호출합니다:

  • 현재 탭의 해석 line만 남기고 다른 .tran/.dc/.ac/.noise는 일시 제거
  • 학생의 .control...endc 블록도 일시 제거 (자동 wrapper로 대체)
  • 자동 wrapper로 결과를 /<analysis>.txt에 dump

예: Tran 탭의 Run 버튼 동작:

* (.dc / .ac / .noise / .control 제거)
.tran <학생이 쓴 시간 파라미터>
.control
run
wrdata /tran.txt all
quit
.endc
.end

3. 해석별 표시

  • Tran: 시간(s) vs V/I (linear)
  • DC: sweep 변수 vs V/I (linear)
  • AC: 주파수(log) vs |H| dB & ∠H deg (Bode plot 2단)
  • Noise: 주파수(log) vs noise spectral density (log-log) — 1/f noise 미포함

4. ngspice 로그

ngspice의 stdout/stderr 전체가 좌측 하단 📜 ngspice 로그에 표시됩니다. 디버깅 시 참고하세요.

5. 예제

헤더의 예제 로드 드롭다운에서 4종 예제를 즉시 로드할 수 있습니다.

6. 제한

  • 실행은 30초 timeout. AC sweep을 너무 광대역으로 잡으면 중단될 수 있습니다.
  • nested .dc sweep은 1차 sweep만 표시됩니다.
  • 한 번에 하나의 ngspice 세션만 실행됩니다 (Stop 버튼으로 중단 가능).
